## 执行摘要

面向内部 QA/开发人员，解决在首页须逐一点击子栏目才能查看内容的效率问题；在每个叶子节点名称下新增水平滚动内容预览条，按需懒加载展示缩略图与名称，使人员无需弹窗即可快速扫描各子栏目数据。

---

## 1. 项目概述

### 1.1 业务背景

**现状描述**:
- OS10-prod-QA-latest.html 以树形结构展示栏目层级，叶子节点须点击后才能在弹窗中查看内容
- 每次查看需要触发接口请求 → 等待 Modal → 阅读 → 关闭，流程繁琐

**痛点分析**:

| 痛点编号 | 痛点描述 | 影响范围 | 量化影响 | 紧迫程度 |
|----------|----------|----------|----------|----------|
| P1 | 无法在首页预览子栏目内容，需逐一点击 | QA/开发日常数据核查 | 每次检查需多 2-3 次操作 | 中 |

**机会识别**:
- 在不破坏现有弹窗流程的前提下，提供快速预览能力，提升首屏信息密度

### 1.2 项目目标

**技术目标**:

| 目标编号 | 目标描述 | 衡量指标 | 目标值 |
|----------|----------|----------|--------|
| TG1 | 首屏加载时间不因预览条增加而变慢 | 初始页面 TTI | 与改动前相比增加 < 200ms |
| TG2 | 预览条图片懒加载，仅在视口内触发请求 | 接口调用数 | 仅视口内节点触发 |

### 1.3 项目范围

**范围内（In Scope）**:
- 叶子节点（无子节点的树节点）下方新增水平滚动内容预览条
- 数据取自 `column/content` 接口的 `dataList`，仅使用 `icon` 和 `name` 字段
- IntersectionObserver 懒加载，节点进入视口时请求数据
- 骨架屏占位，加载完成后渲染
- 空数据显示 "no data"；失败重试一次，仍失败显示 "Failed to load"
- 点击预览项复用现有 contentModal 展示完整内容
- 宽屏（≥992px）可见 6-7 个，窄屏（<768px）可见 4-5 个
- 移动端 touch 手势水平滚动

**范围外（Out of Scope）**:
- 父节点预览条（仅叶子节点，父节点不展示）
- 预览条独立分页/筛选功能
- 引入任何新 JS/CSS 库

**未来规划（Future Scope）**:
- 可考虑对父节点也展示聚合预览（按需评估）

### 1.4 成功标准

| 维度 | 成功标准 | 衡量方式 |
|------|----------|----------|
| 功能完整性 | 所有叶子节点下均出现预览条 | 人工验收 |
| 响应式布局 | 宽屏 6-7 个、窄屏 4-5 个可见 | 浏览器 DevTools 验证 |
| 懒加载正确 | 仅视口内节点触发接口请求 | Network 面板观察 |
| 移动端流畅 | touch 滑动无卡顿 | 移动端实机/模拟器测试 |

### 1.5 技术可行性验证（POC）

| 关键假设 | 验证方法 | 验证结果 | 是否阻塞 |
|----------|----------|----------|----------|
| IntersectionObserver 在目标浏览器可用 | Chrome/Firefox 开发者工具验证 API 存在 | 通过，Chrome 58+/Firefox 55+ 均支持 | 否 |
| `getColumnContent()` 可复用，无需改签名 | 代码审阅（已读取源码第 1110-1139 行） | 通过，函数接受 columnId 返回 Promise | 否 |
| CSS scroll-snap + overflow-x:auto 满足滚动需求 | 无需 POC，标准 CSS | 通过 | 否 |

**判定**: 全部通过，继续后续章节。

## 3. 核心场景与用户故事

### 3.1 场景总览

| 场景编号 | 场景名称 | 优先级 | 涉及角色 | 使用频率 | 业务价值 |
|----------|----------|--------|----------|----------|----------|
| SC-001 | 首页扫描子栏目内容预览 | P0 | QA/开发工程师 | 高 | 核心 |
| SC-002 | 点击预览项查看完整内容 | P1 | QA/开发工程师 | 中 | 重要 |

### 3.2 P0 场景详述

#### SC-001: 首页扫描子栏目内容预览

> **用户故事**: 作为 QA 工程师，我想要在首页直接看到各子栏目的内容缩略图，以便无需逐一点击弹窗即可快速判断数据是否正常。

**前置条件**: 页面已完成 token 获取和栏目树渲染

**主流程步骤**:
1. 页面渲染完成，叶子节点名称下方显示骨架屏占位行
2. 用户滚动，叶子节点进入视口，触发 `getColumnContent(columnId)` 请求
3. 请求返回后，骨架屏替换为水平滚动预览条
4. 预览条展示 icon 图片（原始比例，限制高度）和 name 文字
5. 用户可左右滑动查看更多内容项

**异常流程**:

| 异常编号 | 异常条件 | 系统行为 | 用户提示 |
|----------|----------|----------|----------|
| EX-001 | 接口返回 dataList 为空 | 渲染空状态 | 显示 "no data" |
| EX-002 | 接口请求失败 | 自动重试一次 | 重试仍失败则显示 "Failed to load" |
| EX-003 | 图片 icon URL 加载失败 | 显示默认占位图 | 图片区域显示 1x1 灰色像素 |

**验收标准（AC）**:
- [ ] AC1: 叶子节点名称下方出现内容预览条，父节点不出现
- [ ] AC2: 宽屏（≥992px）一行可见 6-7 个内容项
- [ ] AC3: 窄屏（<768px）一行可见 4-5 个内容项
- [ ] AC4: 预览条仅在节点进入视口时触发请求（Network 面板可验证）
- [ ] AC5: 加载中显示骨架屏占位

### 3.3 P1 场景概述

#### SC-002: 点击预览项查看完整内容

> **用户故事**: 作为 QA 工程师，我想要点击预览条中的某项，以便快速打开该子栏目的完整内容弹窗。

**主流程**: 点击预览条中任意内容项 → 触发现有 contentModal（复用 `getColumnContent` + `renderModalContent` 逻辑）→ 弹窗展示该子栏目完整数据列表

**验收标准**:
- [ ] 点击预览项后打开 contentModal，内容与原有点击节点名称一致

## 4. 功能需求

### 4.1 功能需求清单

| 需求编号 | 功能模块 | 需求描述 | 优先级 | 关联场景 |
|----------|----------|----------|--------|----------|
| FR-001 | 预览条渲染 | 叶子节点名称下方增加水平滚动预览条容器 | P0 | SC-001 |
| FR-002 | 懒加载 | 使用 IntersectionObserver，节点进入视口才请求数据 | P0 | SC-001 |
| FR-003 | 骨架屏 | 加载期间显示灰色矩形占位符 | P0 | SC-001 |
| FR-004 | 内容项渲染 | 展示 icon 图片（原始比例，max-height 限制）+ name 文字 | P0 | SC-001 |
| FR-005 | 空/失败状态 | 空数据显示 "no data"；失败重试一次后显示 "Failed to load" | P0 | SC-001 |
| FR-006 | 点击交互 | 点击预览项复用现有 contentModal 展示完整内容 | P1 | SC-002 |

### 4.2 功能模块划分

- **CSS 模块**: `.subcol-preview-strip`（预览条容器，overflow-x scroll）、`.subcol-preview-item`（内容项）、`.subcol-skeleton`（骨架屏）
- **JS 模块**: `initSubcolPreview(columnId, $container)` 初始化懒加载观察、`renderSubcolPreview($container, dataList)` 渲染内容、`renderSubcolSkeleton($container)` 渲染骨架屏

### 4.3 接口需求

| 接口编号 | 对接系统 | 接口类型 | 数据方向 | 备注 |
|----------|----------|----------|----------|------|
| IF-001 | `/sp/api/device/v1/column/content` | REST GET | 入（读取 dataList | 复用现有 `getColumnContent()` 函数 |
