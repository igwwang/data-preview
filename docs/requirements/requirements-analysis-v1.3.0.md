## 执行摘要

内部研发/QA 人员使用 OS10-prod-QA-latest.html 调试时，需要获取子栏目的完整 API URL 以便在浏览器中直接查看原始响应。当前需手动拼接 URL，效率低。本次在树节点 `#id` 区域增加点击复制功能，一键获取完整 URL，1.5s 视觉反馈确认。

## 1. 项目概述

### 1.1 业务背景

**现状描述**:
- 树视图叶节点展示 `栏目名 #id` 格式，`#id` 仅为展示文本，无交互
- 调试人员需手动打开 DevTools 或拼接 URL 才能访问 column content 接口

**痛点分析**:

| 痛点编号 | 痛点描述 | 影响范围 | 量化影响 | 紧迫程度 |
|----------|----------|----------|----------|----------|
| P1 | 手动拼接 URL 易出错（token 需实时获取），需切换 DevTools | 研发/QA | 每次调试多耗费 30s+ | 中 |

### 1.2 项目目标

**技术目标**:

| 目标编号 | 目标描述 | 衡量指标 | 目标值 | 达成时限 |
|----------|----------|----------|--------|----------|
| TG1 | 点击 #id 区域一键复制完整 URL | 用户验收 | 100% 场景可用 | v1.3.0 |

### 1.3 项目范围

**范围内（In Scope）**:
- 叶节点 `#id` 文本区域绑定 click handler
- 复制 `${API_BASE_URL}/sp/api/device/v1/column/content?token=${globalToken}&columnIds=${item.id}`
- 复制成功后 #id 文本短暂变为 "✓ Copied!"（1.5s 后恢复）
- Clipboard API 不可用时 alert 弹出完整 URL
- `stopPropagation` 阻止冒泡，不触发父级 contentModal 弹窗

**范围外（Out of Scope）**:
- 父节点 #id 区域（无交互需求）
- 其他接口 URL 的复制
- 右键菜单或按钮形式

### 1.4 成功标准

| 维度 | 成功标准 | 衡量方式 | 评估时间点 |
|------|----------|----------|------------|
| 功能 | 点击 #id 剪贴板内容与手动拼接结果一致 | 人工对比验收 | 交付时 |
| 体验 | 1.5s 内看到 "✓ Copied!" 反馈 | 人工测试 | 交付时 |
| 回归 | 原有弹窗主流程正常 | 人工回归 | 交付时 |

### 1.5 技术可行性验证（POC）

| 关键假设 | 验证方法 | 验证结果 | 是否阻塞 |
|----------|----------|----------|----------|
| `navigator.clipboard.writeText()` 在 file:// 协议可用 | 查看现有 `copyPlayUrlBtn` 的实际运行（已上线 v1.1.0） | 通过，已有同模式代码正常运行 | 否 |
| `stopPropagation` 可阻止 node-content 的父级 click | jQuery 事件冒泡机制，标准用法 | 通过，无需 POC | 否 |

**判定**: 全部通过，继续后续章节。

### 3.2 P0 场景详述

#### SC-001: 点击 #id 复制 column content URL

> **用户故事**: 作为 QA 调试人员，我想要点击子栏目的 `#id` 文本，以便直接获取该栏目的完整 API URL 用于浏览器调试。

**前置条件**: 页面已加载完成，`globalToken` 已获取

**主流程步骤**:
1. 用户在树视图中看到叶节点，例如 `Sports #932409818680464025`
2. 用户点击 `#932409818680464025` 文字区域
3. 系统调用 `navigator.clipboard.writeText(url)`，url = `https://saas.zeasn.tv/sp/api/device/v1/column/content?token=<token>&columnIds=932409818680464025`
4. `#id` 文本变为 "✓ Copied!"，1.5s 后恢复为 `#932409818680464025`
5. 用户粘贴 URL 到浏览器地址栏，直接查看响应

**异常流程**:

| 异常编号 | 异常条件 | 系统行为 | 用户提示 |
|----------|----------|----------|----------|
| EX-001 | Clipboard API 不可用（权限被拒或旧浏览器） | catch 块捕获异常 | alert 弹出完整 URL，用户可手动复制 |

**验收标准（AC）**:
- [ ] AC1: 点击叶节点 #id 区域后，剪贴板内容为完整 URL，包含当前 token 和正确 columnId
- [ ] AC2: 复制后 #id 文本在 1.5s 内显示 "✓ Copied!"，之后恢复原始 `#<id>` 文本
- [ ] AC3: 点击 #id 区域不触发 contentModal 弹窗
- [ ] AC4: 点击叶节点名称部分（非 #id）仍正常弹出 contentModal
- [ ] AC5: Clipboard API 失败时 alert 弹出完整 URL
